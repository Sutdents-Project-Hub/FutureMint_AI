# FutureMint AI 決賽 Demo 腳本

## 展示前準備

1. 用 Chrome 開啟 Flutter Web，確認先出現「登入 FutureMint」。
2. 使用專為決賽建立的合成測試帳號登入；不要在台上顯示真實 email、password、token、key 或 Portal。
3. 確認 Functions host 和 `ALLOWED_ORIGINS` 都正常；若沒有網路，改選「以訪客模式繼續」，並先說明資料不會儲存。
4. 投影解析度先測 1440×900；手機版備援可用 375px responsive view。

## 四分鐘主線

### 0:00–0:25　登入與資料界線

「FutureMint 讓每個人只看到自己的預算、紀錄與微課。帳號資料經過後端驗證；若只是想體驗，也可用訪客模式，但資料在離開後會清除。」

登入合成測試帳號，指出左上「已登入」。不展示或口述任何真實帳號資訊。

### 0:25–0:55　問題與首頁

「小恩開始自己管理零用錢和數位訂閱，但多數工具只記帳，不會在他做選擇的當下幫他看懂影響。」

指出首頁的本月安心可用、校外活動基金、近期紀錄與非責備式教練提醒。說明展示資料是合成資料，不是真實學生帳本。

### 0:55–1:45　Quick Capture

進入「記一筆」，輸入：

```text
今天買珍奶 75
```

按「幫我整理」，先停在草稿畫面：

- 指出來源是 Azure AI 或 deterministic demo provider。
- 說明 parse 不等於保存，草稿可改金額、項目與分類。
- 按「確認並記下」後才更新帳本與預算。

加分情境可輸入 `Netflix 390，四個人分` 或 `早餐 65，飲料 40`，展示分帳／多草稿。

### 1:45–2:25　訂閱教練

回首頁按「比較方案」。說明所有價格是合成展示資料，不宣稱即時市場價格；成本與排序由程式計算，AI 只能解釋，使用者仍須確認官方資格和條款。

### 2:25–3:00　個人化微課

進入「學習」，選「先檢查一項固定訂閱」。指出課程只取必要的合成事件摘要，並把反思轉成一個七天內可做到的行動；這不是投資建議。

### 3:00–3:35　FutureSeed

進入「未來」，用預設每月 500 元、5 年、假設 3% 按「開始教育試算」。分開說明投入本金、假設成長、期末可能金額與每年數值；公式是確定性普通年金試算，報酬率是使用者可見假設，不保證結果、不推薦金融商品。

### 3:35–4:00　技術收尾

「Flutter 以 Bearer token 呼叫 Functions。Functions 從登入 session 推導帳號，不接受前端指定 user ID；AI 回覆先過 schema，金額與複利再由確定性程式計算。」

## 降級展示

| 情境 | 畫面與說法 |
|---|---|
| Azure AI timeout／429 | 顯示安全錯誤與重試；不宣稱已解析成功 |
| Cosmos 不可用 | 保留草稿，不顯示保存成功；同 idempotency key 可安全重試 |
| 完全無網路 | 可進訪客模式體驗；資料不會儲存，也不宣稱和登入資料同步 |
| 主機無法啟動 | 使用最後成功的 Flutter Web release build 或預錄流程；口頭說明未連 Azure |

## 不可宣稱

- 不可說已部署 Azure、已完成 production 法遵或已連接真實帳戶。
- 不可把 deterministic 30/30 評估當成 Azure OpenAI 準確率。
- 不可說訂閱價格即時、方案一定符合資格、FutureSeed 是報酬預測。
- 不可展示真實學生姓名、學校、帳號、卡號、password、session token、API key 或 Portal secret。
